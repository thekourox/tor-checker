# 🌐 Tor Proxy Network Manager (for 3x-ui / Xray)

یک پنل تحت وبِ فوق‌العاده قدرتمند و بهینه‌شده برای مدیریت، اسکن و ساخت خودکار شبکه‌ای از پروکسی‌های SOCKS5 بر بستر شبکه Tor. این پروژه به صورت اختصاصی برای استفاده به عنوان **نودهای خروجی (Outbounds)** در پنل‌های Xray/3x-ui طراحی شده است.

این سیستم با ایجاد کانکشن‌های ایزوله، تضمین می‌کند که آی‌پی خروجی سرورِ شما دقیقاً متعلق به همان کشوری باشد که انتخاب کرده‌اید.

---

## ✨ ویژگی‌های کلیدی (نسخه نهایی)

- **🚀 موتور پینگِ شاهکار (TCP Ping):** به جای دانلود فایل که باعث بلاک شدن توسط نودهای تور می‌شد، سیستم فقط یک اتصال صفر-بایتی (TCP Handshake) به `google.com` می‌زند. سرعت آپدیت‌ها در پنل بسیار بالاست و ترافیک مصرفی صفر است.
- **🌍 اعتبارسنجی هوشمند موقعیت (Smart Geo-Validation):** تور بعضی مواقع آی‌پی‌ها را اشتباه لوکیشن‌گذاری می‌کند. ما با استفاده از API کلودفلر (`cloudflare.com/cdn-cgi/trace`) مچِ تور را می‌گیریم! اگر آی‌پی با کشور درخواستی شما مغایرت داشته باشد، سرور به صورت خودکار نابود شده و یک آی‌پی تایید شده‌ی جدید جایگزین می‌شود.
- **📌 روتینگِ ثابت (Sticky Ports):** پورت‌های اختصاص‌یافته به هر کشور مادام‌العمر ثابت می‌مانند. یعنی اگر رول‌های روتینگ را روی 3x-ui تنظیم کنید، با ری‌استارت شدن سرور، پورت‌ها به هم نمی‌ریزند.
- **⚙️ هوش تشخیص سخت‌افزار (Auto-Provisioning):** اسکریپت به صورت خودکار رم و CPU سرور شما را اسکن کرده و بهترین محدودیت‌ها را (از جمله رم مصرفی هر نود تور، بازه پینگ‌گیری و تعداد کارگرها) روی سیستم اعمال می‌کند.
- **🔄 تعویض زنده (Zero-Downtime Reload):** می‌توانید در حالی که سرور در حال کار است، کشورها را از تنظیمات حذف یا اضافه کنید. سیستم بدون قطع کردن ترافیکِ کشورهای در حال کار، فقط کشورهای جدید را می‌سازد.
- **💻 دیمن‌های پس‌زمینه (Daemons):** اجرای کاملاً ایزوله در بک‌گراند بدون نیاز به باز ماندن ترمینال SSH در لینوکس یا CMD در ویندوز.

## 🖥️ مدیریت منابع و سخت‌افزار (بسیار مهم)

شبکه تور بسیار سنگین است و هر کشوری که انتخاب می‌کنید، یک پروسه‌ی (Process) کاملاً مستقل در سرور شما باز می‌کند که رم و پردازنده مصرف می‌کند. **لطفاً طمع نکنید!** اگر محدودیت‌های سخت‌افزاری را رعایت نکنید، سرور شما کِرَش (Crash) خواهد کرد.

### 🐢 سرورهای ضعیف (۱ الی ۲ هسته / ۱ الی ۴ گیگ رم)
- **حداکثر کشورها:** ۱۰ الی ۱۵ کشور
- **توضیحات:** به هیچ عنوان روی یک سرور با ۴ گیگابایت رم، ۶۰ کشور را انتخاب نکنید! تنظیمات هوشمند پنل روی این سرورها پینگ را هر ۶۰ ثانیه می‌گیرد تا CPU شما آزاد بماند.

### 🚀 سرورهای متوسط (۴ هسته / ۴ الی ۸ گیگ رم)
- **حداکثر کشورها:** ۳۰ الی ۵۰ کشور
- **توضیحات:** ایده‌آل‌ترین حالت برای یک پنلِ پرسرعت. سیستم روی این کانفیگ پینگ‌ها را هر ۳۰ ثانیه می‌گیرد و محدودیت رمِ هر پروسه را کمی آزادتر می‌کند تا سرعت ترافیک عبوری بالاتر برود.

### ⚡ سرورهای قدرتمند (۸ هسته به بالا / ۱۶ گیگ رم به بالا)
- **حداکثر کشورها:** ۱۰۰ الی ۱۹۰ کشور
- **توضیحات:** هیولای پروکسی! با این سخت‌افزار می‌توانید سیستم را روی بازه پینگ‌گیری ۱۵ ثانیه بگذارید و محدودیت پهنای باند را روی صفر (نامحدود) تنظیم کنید. کارگرهای زمان‌بند (Scheduler Workers) تمام هسته‌های شما را به کار می‌گیرند.

## 🛠️ آموزش نصب و راه‌اندازی

### در محیط لینوکس (سرورهای لایو Ubuntu / Debian)
برای اینکه اسکریپت به صورت دائم روشن بماند (حتی بعد از خروج از SSH)، از فایل نصبی که برای شما آماده شده استفاده کنید:
1. ابتدا به مسیر پروژه بروید: `cd /path/to/tor-checker`
2. دستور نصب سیستم را اجرا کنید: `sudo bash install_service.sh`
3. سیستم به صورت سرویس `systemd` در پس‌زمینه استارت می‌خورد. حالا با خیال راحت از سرور خارج شوید!
- **برای بررسی لاگ‌ها:** `journalctl -u tor-checker -f`
- **برای خاموش کردن کامل:** `systemctl stop tor-checker`

### در محیط ویندوز (برای لوکال دولوپمنت)
1. برای اجرای مخفیانه (بدون باز شدن صفحه سیاه)، روی فایل **`start_background.vbs`** دابل کلیک کنید. 
2. برای خاموش کردن کامل تمامی سرویس‌های تور و پایتون، روی **`stop_background.bat`** دابل کلیک کنید.

*(برای دسترسی به پنل، در مرورگر خود آدرس `http://IP-SERVER:8000` را باز کنید)*
